Ways to Implement Preprocessing

CSS preprocessing can be implemented through the use of either LESS or SASS. LESS and SASS are used to make stylesheets more organized and easier to navigate on the client side. This is accomplished through the use of variables, mixins, and nesting. Both LESS and SASS can be deployed to a staging server to see updates right away.

Mechanisms of SASS

  • Variables in SASS are a way to store information (in this case CSS values) that can then be used throughout the stylesheet. The symbol $ makes something a variable.
  • Nesting in SASS allows CSS selectors to follow a similarly nested heiarchy to that of HTML. Selectors can be nested inside other selectors.
  • Mixins in SASS are similar to variables in that they are CSS declarations which can be written once and then reused by calling the mixin.
  • Operators in SASS allow use of simple math operations within CSS in order to calculate different values to be used in the resulting stylesheet.
